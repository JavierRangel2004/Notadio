  # Notadio Full-Page Premium Brand Board Rebrand

  ## Summary

  Rebuild the entire Notadio page as a single dark premium brand-system board that
  also remains the working product UI. Use the attached reference as a strong visual
  model, not a literal clone: same high-end art-direction, exact palette, glassy dark
  surfaces, modular investor-deck composition, and premium AI-tool feel, but adapted
  to real upload, processing, transcript, summary, translation, and diarization
  workflows.

  This implementation is frontend-only unless a gap is discovered during execution.
  No backend/API changes are planned. Existing functionality stays intact; the page
  shell, layout hierarchy, styling system, and presentation of product states are
  what change.

  ## Implementation Changes

  ### Visual system

  - Replace the current warm editorial system with a dark premium system built around
    these exact base colors:
      - #0A0510 page background
      - #1A0B2E secondary surface
      - #F8F9FA primary text
      - #A09DB0 muted text
      - #9E1B32 primary accent
      - #5D2A7A secondary accent
  - Derive only supporting tints from those colors for borders, hover states, focus
    rings, translucent glass layers, waveform glow, and disabled states.
  - Use a 12-column desktop grid and a strict modular card system so the page reads
    like a creative director presentation board, not a generic dashboard.
  - Use Sora for headings/display and Manrope for body/UI text. Use IBM Plex Mono
    only for timestamps, technical badges, and diagnostics.
  - Keep all filenames in the UI font, never the display font.

  ### Full-page composition

  - Convert the full page into one continuous board with these permanent sections:
      - Brand concepts rail: Notadio wordmark, icon/fav-icon concepts, brand
        keywords.
      - Palette and typography module.
      - Website UI concepts module: hero, feature cards, transcript dashboard,
        workspace/library, footer preview.
      - Components module: nav, primary buttons, diarization tags, waveform/audio
        controls, tooltips, status pills.
      - Motion language module: static visual treatment showing pulse, waveform glow,
        translation transitions, dashboard snap.
      - Product mood/reference tiles: abstract sound-to-text, AI fiber optics,
        focused workspaces.
  - Do not make this a separate “marketing-only” sheet. The board is the page
    framework itself.

  ### State mapping to the real app

  - Idle/upload state:
      - Use the reference-board composition most literally here.
      - Left/top modules present brand identity, color system, keywords, and
        miniature UI previews.
      - The actual upload interaction becomes the “primary action” centerpiece and is
        styled like the board’s hero mockup.
  - Processing state:
      - Keep the same page shell and modules, but convert the central hero/UI concept
        area into a live processing dashboard.
      - Progress, logs, runtime profile, and current stage sit inside high-contrast
        glass cards with crimson focus states and subtle waveform/recording glow.
  - Completed/results state:
      - Keep the board shell intact.
      - The “website UI concepts” and “feature showcase” areas become the live
        summary + transcript workspace.
      - The summary rail stays art-directed, but the transcript area becomes a dense,
        practical tool surface.
      - Export, language switch, summary regenerate, and speaker-ID rerun controls
        move into a dedicated premium control strip.

  ### Product-specific content mapping

  - Replace generic board placeholders with real Notadio modules:
      - Summaries
      - Translation
      - Diarization
      - Transcript review
      - Workspace/library
  - Build the nav bar as a real top-level glass navigation with brand mark, utility
    controls, and app actions.
  - Build the “feature showcase” as actual product cards that reflect current Notadio
    capabilities.
  - Build the waveform/audio-control component as a styled static product surface
    unless existing audio playback exists.
  - Speaker tags must use premium dark chips with clear contrast and categorical
    color separation.

  ### Transcript and summary presentation

  - Keep grouped transcript rendering as the default.
  - Style each speaker group as a premium glass panel with:
      - start/end time block
      - speaker tag
      - grouped utterances
      - strong spacing and readable line height
  - Summary area should visually match the board’s premium showcase modules while
    keeping all existing structured fields:
      - headline
      - brief
      - sections
      - key decisions
      - action items
      - follow-ups
      - risks
      - operational notes
      - open questions
  - Action items become a “tasks and todos” board using crimson-accented cards with
    owner/deadline/status metadata.

  ### Accessibility and UX requirements

  - Treat accessibility as a hard constraint, not a later polish step.
  - Maintain WCAG AA contrast for normal text and controls on dark surfaces.
  - Add visible :focus-visible states for all buttons, links, segmented controls,
    file inputs, details/summary elements, and export actions.
  - Add reduced-motion fallbacks for pulse/glow/transition effects.
  - Use clear hover, active, disabled, and selected states with both color and shape/
    border changes.
  - Preserve full filenames with wrap-safe rules and title attributes.
  - Ensure the mobile layout is intentional:
      - one-column stack
      - control strips become grouped rows
      - transcript remains readable
      - no clipped board modules

  ### Files to change

  - frontend/src/App.tsx
    Recompose the page structure and map the board sections to live product state.
  - frontend/src/styles.css
    Replace the current visual system with the premium dark board system, including
    responsive layout, accessibility states, and component styling.
  - frontend/src/api.ts
    No planned interface changes; only touch if type imports or frontend rendering
    needs cleanup.

  ## Test Plan

  - Build checks:
      - npm --workspace frontend run build
      - npm --workspace backend run build
  - Visual checks on desktop:
      - idle/upload board
      - processing board
      - completed/results board
      - failed state
  - Responsive checks:
      - 1440px
      - 1024px
      - 768px
      - 390px
  - Accessibility checks:
      - keyboard-only navigation across upload, segmented controls, exports, retry
        actions, logs
      - visible focus ring everywhere interactive
      - reduced-motion mode
      - contrast review for muted text, pills, warnings, buttons, transcript metadata
  - Content stress checks:
      - very long filename with mojibake-repaired accents
      - very long summary with many sections and many action items
      - transcript with many speakers
      - no-summary / empty-summary state
      - no-english-translation state

  ## Assumptions

  - The attached image is the art-direction source, but implementation should remain
    product-usable and not degrade workflow clarity.
  - No new routes or backend endpoints are needed.
  - Logo/icon concepts will be implemented as inline SVG/CSS presentation concepts on
    the page, not as final exported brand assets.
  - Derived tints/shadows are allowed, but the six specified base colors remain the
    primary palette.
  - Accessibility wins over visual mimicry if the reference aesthetic conflicts with
    contrast or interaction clarity.
